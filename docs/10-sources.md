# Sources

Annotated bibliography for the "loop engineering" methodology of Boris Cherny and Cat Wu. Sources are grouped into **primary** (Boris Cherny, Cat Wu, or Anthropic speaking directly: their own posts, official transcripts, Anthropic blog/docs) and **secondary** (third-party coverage, aggregators, and commentary). A short note on each entry describes what it provides and flags any sourcing caveats.

A recurring caveat applies across the loop material: the verified-primary origin of the "I write loops" quote is Boris Cherny's own on-camera remarks (Sequoia AI Ascent 2026, the Acquired Unplugged / WorkOS talk on June 2, 2026, and the Anthropic "one year of Claude Code" video). The named term "loop engineering" itself is a third-party coinage (Addy Osmani, Peter Steinberger, and journalists), not Anthropic's or Cherny's own label.

- [Run Claude Code programmatically (headless / claude -p) — Claude Code Docs](https://code.claude.com/docs/en/headless) — `claude -p`/`--print`, `--output-format text|json|stream-json`, `--json-schema`, `--bare`, `--resume`/`--continue`, session_id capture, `--allowedTools`, `--append-system-prompt`. Core grounding for the loop harness.
- [Agent SDK overview — Claude Code Docs](https://code.claude.com/docs/en/agent-sdk/overview) — "Same tools, agent loop, and context management that power Claude Code." Python/TypeScript packages, `query()`, subagents, hooks, MCP, sessions, permissions. The productized way to write loops (formerly the Claude Code SDK).
- [Keep Claude working toward a goal (/goal) — Claude Code Docs](https://code.claude.com/docs/en/goal) — The built-in condition-driven loop: works turn after turn until a small fast model confirms the condition. A wrapper around a prompt-based Stop hook; works headless. Strongest grounding for "iterate until a stopping condition."
- [Run prompts on a schedule (/loop, Cron tools) — Claude Code Docs](https://code.claude.com/docs/en/scheduled-tasks) — The bundled `/loop` skill: fixed interval, self-paced, bare maintenance prompt, `loop.md` customization, 7-day expiry, 50-task cap, Esc to stop. Compares cloud routines, desktop tasks, and `/loop`; also documents the Monitor tool.
- [Create custom subagents — Claude Code Docs](https://code.claude.com/docs/en/sub-agents) — Subagents run in isolated context windows with their own system prompt, tools, and permissions; preserve context and control cost (e.g. Haiku). Defined in `.claude/agents`; `isolation: worktree` frontmatter.
- [Extend Claude with skills (SKILL.md) — Claude Code Docs](https://code.claude.com/docs/en/skills) — Skills are `.claude/skills/<name>/SKILL.md`, loaded only when used; custom commands merged into skills; the Agent Skills open standard; invocable via `/name`.
- [How Claude remembers your project (CLAUDE.md + auto memory) — Claude Code Docs](https://code.claude.com/docs/en/memory) — CLAUDE.md scopes, `@imports`, `.claude/rules/` path-scoping, auto memory under `~/.claude/projects/<project>/memory/`. Notes CLAUDE.md is context, not enforcement; use PreToolUse hooks to enforce.
- [Hooks in Claude Code — Claude Code Docs](https://code.claude.com/docs/en/hooks) — Lifecycle events (PreToolUse/PostToolUse/Stop/SubagentStop/SessionStart/UserPromptSubmit/WorktreeCreate, etc.), handler types, exit-code decision control. A Stop hook is custom loop termination; `/goal` wraps a prompt-based Stop hook.
- [Run parallel sessions with worktrees — Claude Code Docs](https://code.claude.com/docs/en/worktrees) — `claude --worktree`/`-w`, `.claude/worktrees/<value>/`, EnterWorktree/ExitWorktree tools, `isolation: worktree` subagents, `.worktreeinclude`, `worktree.baseRef`. The correct path is `/worktrees` (`/git-worktrees` 404s).
- [Manage multiple agents with agent view — Claude Code Docs](https://code.claude.com/docs/en/agent-view) — `claude agents` fleet UI: background sessions grouped Needs input / Working / Completed; auto-moves each dispatched session into its own worktree. Grounds the parallel-agents/fleet workflow.
- [Connect Claude Code to tools via MCP — Claude Code Docs](https://code.claude.com/docs/en/mcp) — MCP servers via `.mcp.json`, `mcp__<server>__<tool>` namespacing, examples (JIRA, Postgres, Figma, Gmail, Playwright). The loop's external read/write layer.
- [Push events into a running session with channels — Claude Code Docs](https://code.claude.com/docs/en/channels) — Research preview. An MCP server pushes events into a running session via `--channels`; the event-driven alternative to `/loop` polling. Telegram/Discord/iMessage/fakechat. v2.1.80+, not on Bedrock/Vertex/Foundry.

## Disputed / unresolved attribution

Multiple secondary sources place the canonical "I write loops" quote at different venues: Sequoia AI Ascent 2026 (the firmest first-party origin per the verification verdicts), the Acquired Unplugged / WorkOS talk (June 2, 2026), the Anthropic "one year of Claude Code" video (~June 8, 2026), and, less reliably, Lenny's Podcast.
